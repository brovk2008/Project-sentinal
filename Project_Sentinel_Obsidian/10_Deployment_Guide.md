# 10 — Deployment Guide

This document describes the step-by-step setup and deployment instructions for Project Sentinel.

---

## 1. Port Allocation Map

Ensure the following ports are open and not occupied by other services before starting:

| Service | Host Port | Protocol | Config Location |
|---|---|---|---|
| **FastAPI Backend (Local)** | `9000` / `8000` | HTTP | `backend/main.py` |
| **React Frontend (Local)** | `5173` | HTTP | `frontend/package.json` |
| **Zoho Catalyst AppSail (Prod)** | Dynamic (`PORT`) | HTTPS | `backend/app-config.json` |

---

## 2. Step-by-Step Installation & Deployment

### Step A: Local Development & Fallback Mode
For quick local audits and testing, Project Sentinel operates in a localized standalone mode:
1. Create `.env` from `.env.example` in the project root:
   ```bash
   copy .env.example .env
   ```
2. The backend automatically detects the presence of local configs and falls back to:
   - **Database**: Local SQLite replica at `backend/sentinel_local.db`.
   - **RAG Embeddings**: SentenceTransformers local fallback using Hugging Face's inference API.

### Step B: Python Virtual Environment
1. Create and activate a Python virtual environment:
   ```bash
   python -m venv venv
   # On Windows:
   venv\Scripts\activate
   ```
2. Install the necessary data processing, ML, and RAG dependencies:
   ```bash
   pip install -r backend/requirements.txt
   ```

### Step C: Local ML & Embedding Diagnostics
1. Verify that `backend/models/metrics.json` exists (generated by the model training pipeline).
2. Run model validation locally to ensure prediction models are functional:
   ```bash
   python backend/ml/train_all.py
   ```

### Step D: Zoho Catalyst AppSail Deployment
To deploy Project Sentinel into the live production environment on Zoho Catalyst:
1. **Install Zoho Catalyst CLI**:
   ```bash
   npm install -g zcatalyst-cli
   ```
2. **Log in to Catalyst**:
   ```bash
   catalyst login
   ```
3. **Verify the active project**:
   Ensure the CLI is mapped to `Project-Sentinel` (ID `50170000000013047`):
   ```bash
   catalyst project:list
   ```
4. **Configure Production Environment Variables**:
   In the Zoho Catalyst console, navigate to your AppSail service environment configurations and set the following variables:
   - `GROQ_API_KEY`: API authentication key for Groq Cloud completions.
   - `HF_TOKEN`: Hugging Face access token for authenticated query embeddings.
   - `ENV`: `production`
5. **Deploy the application**:
   Compile the frontend assets, zip the backend, and push the service to AppSail:
   ```bash
   # Build frontend static files
   cd frontend && npm install && npm run build && cd ..
   # Deploy only the backend service to AppSail
   catalyst deploy --only appsail:sentinel-backend
   ```

### Step E: Verify Production Deployment
After deployment completes, check the live AppSail URLs returned by the CLI:
* **Production URL**: `https://sentinel-backend-50042879481.development.catalystappsail.in`
* Verify endpoints:
  - Root: `GET /`
  - Health: `GET /api/v1/intelligence/health`
  - Diagnostics: `GET /api/v1/diagnostic/db`
  - Embeddings: `GET /api/v1/diagnostic/embeddings`
