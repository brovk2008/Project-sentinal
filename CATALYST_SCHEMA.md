# Zoho Catalyst Datastore Schema Design - Project Sentinel

This document defines the Zoho Catalyst Datastore table configurations, field names, data types, indexes, and relationships designed to replace the Supabase PostgreSQL database.

---

## 1. RAG & Document Embeddings

### Table: `rag_document_embeddings`
Stores parsed PDF document text chunks and precomputed 384-dimensional semantic embeddings.

| Column Name | Catalyst Data Type | Constraints | Description |
| --- | --- | --- | --- |
| `ROWID` | BigInt | Primary Key (Auto) | Unique row identifier generated by Catalyst. |
| `chunk_id` | Integer | Unique | Unique identifier from Supabase. |
| `document_name` | VarChar(255) | Mandatory | Filename of the source PDF. |
| `page_number` | Integer | Mandatory | Page number in the source PDF. |
| `text_content` | VarChar(4000) | Mandatory | Chunked text content. |
| `metadata_json` | VarChar(1000) | Optional | JSON string of metadata attributes. |
| `embedding` | VarChar(4000) | Mandatory | Stringified JSON array containing 384 float values. |

*   **Indexes**: Unique index on `chunk_id`. Index on `document_name`.

## 2. Lookup & Metadata Tables

### Table: `dim_crime_classification`
Categorizes major crime groups and sub-heads.

| Column Name | Catalyst Data Type | Constraints | Description |
| --- | --- | --- | --- |
| `ROWID` | BigInt | Primary Key (Auto) | Unique identifier. |
| `crime_class_id` | Integer | Unique, Mandatory | Unique classification ID. |
| `crime_group_name` | VarChar(255) | Mandatory | Major crime group (e.g. Cyber Crime). |
| `crime_head_name` | VarChar(255) | Mandatory | Sub-classification name. |

*   **Indexes**: Unique index on `crime_class_id`. Composite index on `(crime_group_name, crime_head_name)`.

### Table: `dim_police_units`
Information on police units and stations.

| Column Name | Catalyst Data Type | Constraints | Description |
| --- | --- | --- | --- |
| `ROWID` | BigInt | Primary Key (Auto) | Unique identifier. |
| `unit_id` | Integer | Unique, Mandatory | Police unit identifier. |
| `unit_name` | VarChar(255) | Mandatory | Police station/office name. |
| `district_name` | VarChar(255) | Mandatory | District parent unit. |
| `circle_name` | VarChar(255) | Optional | Circle/sub-division name. |
| `latitude` | Double | Optional | Latitude coordinate. |
| `longitude` | Double | Optional | Longitude coordinate. |

*   **Indexes**: Unique index on `unit_id`. Index on `district_name`.

### Table: `district_centroids`
Lookup of geographic centroids for Karnataka districts.

| Column Name | Catalyst Data Type | Constraints | Description |
| --- | --- | --- | --- |
| `ROWID` | BigInt | Primary Key (Auto) | Unique identifier. |
| `district_name` | VarChar(255) | Unique, Mandatory | Name of the district. |
| `latitude` | Double | Mandatory | Center latitude. |
| `longitude` | Double | Mandatory | Center longitude. |

*   **Indexes**: Unique index on `district_name`.

### Table: `dim_geography`
Administrative geographic details.

| Column Name | Catalyst Data Type | Constraints | Description |
| --- | --- | --- | --- |
| `ROWID` | BigInt | Primary Key (Auto) | Unique identifier. |
| `geo_id` | VarChar(255) | Unique, Mandatory | Administrative identifier (e.g. DISTRICT_BANGALORE). |
| `district_name` | VarChar(255) | Mandatory | District name. |
| `sub_district_name`| VarChar(255) | Optional | Sub-district/taluk name. |
| `beat_name` | VarChar(255) | Optional | Police beat name. |
| `village_area_name`| VarChar(255) | Optional | Village or urban ward name. |
| `geom_wkt` | VarChar(4000) | Optional | Well-Known Text (WKT) boundary description. |

*   **Indexes**: Unique index on `geo_id`. Index on `district_name`.

### Table: `dim_demographics`
Socio-economic indicators linked to geographies.

| Column Name | Catalyst Data Type | Constraints | Description |
| --- | --- | --- | --- |
| `ROWID` | BigInt | Primary Key (Auto) | Unique identifier. |
| `geo_id` | VarChar(255) | Unique, Mandatory | Foreign Key referencing `dim_geography(geo_id)`. |
| `population_total` | BigInt | Optional | Total population (Census 2011). |
| `population_urban` | BigInt | Optional | Urban population. |
| `literacy_rate` | Double | Optional | Literacy rate percentage. |
| `consumption_index`| Double | Optional | Consumption wealth index. |
| `facebook_wealth_index` | Double | Optional | Relative wealth index. |

*   **Indexes**: Unique index on `geo_id`.

## 3. Crime & Trend Tables

### Table: `fact_fir_events`
Stores detailed First Information Reports. (Seeded with representative subset of ~5,000 rows).

| Column Name | Catalyst Data Type | Constraints | Description |
| --- | --- | --- | --- |
| `ROWID` | BigInt | Primary Key (Auto) | Unique identifier. |
| `fir_id` | VarChar(255) | Unique, Mandatory | Composite unique key. |
| `fir_number` | VarChar(255) | Mandatory | Official crime register ID. |
| `unit_id` | Integer | Mandatory | ID of handling police station. |
| `geo_id` | VarChar(255) | Optional | Geography administrative key. |
| `crime_class_id` | Integer | Mandatory | Crime classification ID. |
| `fir_date` | DateTime | Mandatory | Date/Time crime was registered. |
| `offence_duration_minutes` | Integer | Optional | Calculated duration of offence. |
| `fir_type` | VarChar(255) | Optional | Heinous / Non-Heinous. |
| `fir_stage` | VarChar(255) | Optional | Under Investigation, Closed, etc. |
| `complaint_mode` | VarChar(255) | Optional | Written, Oral, Online. |
| `io_name` | VarChar(255) | Optional | Investigation Officer. |
| `io_kgid` | VarChar(255) | Optional | IO Service ID. |
| `victim_count` | Integer | Optional | Number of victims. |
| `accused_count` | Integer | Optional | Number of accused. |
| `arrested_count` | Integer | Optional | Number of arrested. |
| `conviction_count` | Integer | Optional | Number of convicted. |
| `latitude` | Double | Optional | Precise occurrence latitude. |
| `longitude` | Double | Optional | Precise occurrence longitude. |

*   **Indexes**: Unique index on `fir_id`. Index on `fir_date`. Index on `unit_id`.

### Table: `mv_district_profile`
Pre-aggregated district KPIs to optimize rendering speed.

| Column Name | Catalyst Data Type | Constraints | Description |
| --- | --- | --- | --- |
| `ROWID` | BigInt | Primary Key (Auto) | Unique identifier. |
| `district_name` | VarChar(255) | Unique, Mandatory | District name. |
| `total_firs` | BigInt | Mandatory | Total crimes. |
| `station_count` | Integer | Mandatory | Active police stations. |
| `unique_crime_types` | Integer | Mandatory | Distinct crime categories. |
| `total_victims` | BigInt | Mandatory | Sum of victims. |
| `total_accused` | BigInt | Mandatory | Sum of accused suspects. |
| `total_arrested` | BigInt | Mandatory | Sum of arrested suspects. |
| `earliest_fir` | Date | Optional | Date of oldest FIR. |
| `latest_fir` | Date | Optional | Date of newest FIR. |

*   **Indexes**: Unique index on `district_name`.

### Table: `mv_monthly_trends`
Monthly trend aggregation.

| Column Name | Catalyst Data Type | Constraints | Description |
| --- | --- | --- | --- |
| `ROWID` | BigInt | Primary Key (Auto) | Unique identifier. |
| `month` | Date | Mandatory | Month of the trend. |
| `district_name` | VarChar(255) | Mandatory | District name. |
| `crime_group_name` | VarChar(255) | Mandatory | Crime group name. |
| `fir_count` | BigInt | Mandatory | Total crimes in month. |
| `total_victims` | BigInt | Mandatory | Victims count in month. |
| `total_accused` | BigInt | Mandatory | Accused count in month. |
| `total_arrested` | BigInt | Mandatory | Arrests in month. |
| `total_convicted` | BigInt | Mandatory | Convictions in month. |

*   **Indexes**: Unique composite index on `(month, district_name, crime_group_name)`. Index on `(district_name, crime_group_name)`.

### Table: `mv_station_profile`
Individual station aggregate profile parameters.

| Column Name | Catalyst Data Type | Constraints | Description |
| --- | --- | --- | --- |
| `ROWID` | BigInt | Primary Key (Auto) | Unique identifier. |
| `unit_id` | Integer | Unique, Mandatory | Station unit ID. |
| `station_name` | VarChar(255) | Mandatory | Station name. |
| `district_name` | VarChar(255) | Mandatory | District name. |
| `latitude` | Double | Optional | Station latitude. |
| `longitude` | Double | Optional | Station longitude. |
| `fir_count` | BigInt | Mandatory | Total FIRs. |
| `victims` | BigInt | Mandatory | Total victims. |
| `accused` | BigInt | Mandatory | Total accused. |
| `arrests` | BigInt | Mandatory | Total arrests. |
| `convictions` | BigInt | Mandatory | Total convictions. |
| `top_crime_group` | VarChar(255) | Optional | Dominant crime group. |

*   **Indexes**: Unique index on `unit_id`. Index on `district_name`.

## 4. Anomaly & Financial Networks

### Table: `dim_financial_accounts`
Bank accounts catalog. (Seeded with representative subset of ~2,000 rows).

| Column Name | Catalyst Data Type | Constraints | Description |
| --- | --- | --- | --- |
| `ROWID` | BigInt | Primary Key (Auto) | Unique identifier. |
| `account_number` | VarChar(255) | Unique, Mandatory | Hashed bank account ID. |
| `owner_name` | VarChar(255) | Optional | Owner name. |
| `bank_name` | VarChar(255) | Optional | Bank branch name. |
| `risk_score` | Double | Optional | Isolation Forest fraud risk level. |

*   **Indexes**: Unique index on `account_number`. Index on `risk_score`.

### Table: `fact_financial_transactions`
Individual money transactions. (Seeded with representative subset of ~5,000 rows).

| Column Name | Catalyst Data Type | Constraints | Description |
| --- | --- | --- | --- |
| `ROWID` | BigInt | Primary Key (Auto) | Unique identifier. |
| `transaction_id` | VarChar(255) | Unique, Mandatory | Transaction code. |
| `timestamp` | DateTime | Mandatory | Date/Time of transaction. |
| `sender_account` | VarChar(255) | Mandatory | Source account number. |
| `receiver_account` | VarChar(255) | Mandatory | Destination account number. |
| `amount` | Double | Mandatory | Currency volume. |
| `transaction_type` | VarChar(255) | Optional | CASH_OUT, TRANSFER, etc. |
| `is_fraud` | Boolean | Mandatory | Suspicious transaction label. |
| `velocity_score` | Double | Optional | Transaction velocity score. |
| `geo_anomaly_score` | Double | Optional | Transaction location anomaly score. |

*   **Indexes**: Unique index on `transaction_id`. Index on `sender_account`. Index on `receiver_account`.

### Table: `fact_call_detail_records`
Suspect communication logs.

| Column Name | Catalyst Data Type | Constraints | Description |
| --- | --- | --- | --- |
| `ROWID` | BigInt | Primary Key (Auto) | Unique identifier. |
| `cdr_id` | Integer | Unique, Mandatory | Call detail row ID. |
| `caller_number` | VarChar(255) | Mandatory | Initiator phone. |
| `receiver_number` | VarChar(255) | Mandatory | Receiver phone. |
| `caller_company` | VarChar(255) | Optional | Network provider (Caller). |
| `receiver_company` | VarChar(255) | Optional | Network provider (Receiver). |
| `call_timestamp` | DateTime | Mandatory | Timestamp of call. |
| `duration_seconds` | Integer | Mandatory | Duration of call. |
| `cell_tower_id` | VarChar(255) | Optional | Cellular tower geographical code. |

*   **Indexes**: Unique index on `cdr_id`. Index on `caller_number`. Index on `receiver_number`.

### Table: `mv_network_stats`
Summary dashboard KPIs for networks.

| Column Name | Catalyst Data Type | Constraints | Description |
| --- | --- | --- | --- |
| `ROWID` | BigInt | Primary Key (Auto) | Unique identifier. |
| `total_fraud_transactions` | BigInt | Mandatory | Total fraud counts. |
| `total_fraud_amount` | Double | Mandatory | Total fraud amount (INR). |
| `total_cdr_records` | BigInt | Mandatory | Total call logs. |
| `unique_callers` | BigInt | Mandatory | Total distinct numbers. |
| `high_risk_accounts` | BigInt | Mandatory | Suspect accounts count (>0.7 score). |

*   **Indexes**: Unique index on `total_fraud_transactions`.

### Table: `mv_fraud_graph_edges`
Pre-aggregated fraud nodes/edges connections.

| Column Name | Catalyst Data Type | Constraints | Description |
| --- | --- | --- | --- |
| `ROWID` | BigInt | Primary Key (Auto) | Unique identifier. |
| `source` | VarChar(255) | Mandatory | Origin account number. |
| `target` | VarChar(255) | Mandatory | Destination account number. |
| `transaction_count` | BigInt | Mandatory | Number of fraud transactions. |
| `total_amount` | Double | Mandatory | Sum of money transferred. |
| `sender_risk` | Double | Optional | Risk of source account. |
| `receiver_risk` | Double | Optional | Risk of target account. |
| `sender_name` | VarChar(255) | Optional | Source account owner. |
| `receiver_name` | VarChar(255) | Optional | Destination account owner. |
| `sender_bank` | VarChar(255) | Optional | Source account bank. |
| `receiver_bank` | VarChar(255) | Optional | Destination account bank. |

*   **Indexes**: Unique composite index on `(source, target)`.

### Table: `mv_anomaly_financial`
Highest risk individual financial transactions.

| Column Name | Catalyst Data Type | Constraints | Description |
| --- | --- | --- | --- |
| `ROWID` | BigInt | Primary Key (Auto) | Unique identifier. |
| `transaction_id` | VarChar(255) | Unique, Mandatory | Reference transaction ID. |
| `timestamp` | DateTime | Mandatory | Occurrence timestamp. |
| `sender_account` | VarChar(255) | Mandatory | Source account. |
| `receiver_account` | VarChar(255) | Mandatory | Destination account. |
| `amount` | Double | Mandatory | Amount in INR. |
| `transaction_type` | VarChar(255) | Optional | CASH_OUT, TRANSFER, etc. |
| `is_fraud` | Boolean | Mandatory | Fraud flag. |
| `velocity_score` | Double | Mandatory | Speed score. |
| `geo_anomaly_score` | Double | Mandatory | Location score. |
| `combined_score` | Double | Mandatory | Sum of velocity and location score. |

*   **Indexes**: Unique index on `transaction_id`. Index on `combined_score`.
